Broken Logic - Interactive Testing Guide

Click on each vulnerability category to explore common logic flaws in web applications. Perfect for CTF challenges and bug hunting!

Price Manipulation

Check if prices can be modified client-side

Tests to try:

  • Intercept checkout request and modify price to 0 or negative
  • Change quantity to negative numbers
  • Modify item prices in cart before checkout
  • Use discount codes multiple times or stack them
Example: POST /checkout {"total": 0, "items": [...]}

Cart Session Issues

Manipulate cart session or user's cart

Tests to try:

  • Access another user's cart by changing session/cart ID
  • Add items to cart without authentication
  • Checkout with empty cart but modified total
  • Reuse old cart IDs or session tokens
Example: Change cartId=123 to cartId=456 in requests

Authentication Bypass

Access restricted features without proper auth

Tests to try:

  • Access /checkout or /cart without being logged in
  • Register with admin/special usernames
  • SQL injection in login (admin'--)
  • Check if JWT tokens can be manipulated
Example: username: admin'-- password: anything

IDOR Vulnerabilities

Access other users' data via ID manipulation

Tests to try:

  • Change user_id in profile/order requests
  • Access /api/orders/1, /api/orders/2, etc.
  • Modify order_id in GET/POST requests
  • View other users' order history
Example: GET /api/user/2/orders (access user 2's orders)

Race Conditions

Exploit timing issues in cart/checkout

Tests to try:

  • Apply discount code multiple times simultaneously
  • Checkout same cart twice in parallel requests
  • Modify cart while checkout is processing
  • Use stored credit multiple times
Example: Send 10 parallel checkout requests with same cart

Quantity Manipulation

Exploit math in order calculations

Tests to try:

  • Set quantity to 0 but keep items in cart
  • Use floating point for quantity (1.5 pizzas)
  • Overflow total with huge numbers
  • Integer overflow with max quantity values
Example: POST {"item_id": 1, "quantity": 999999999}

Pro Tips

  • Use browser DevTools Network tab to inspect API requests
  • Try Burp Suite or OWASP ZAP to intercept and modify requests
  • Look for client-side price calculations or validation
  • Test SQL injection in all input fields (login, register, search)
  • Check if authorization is properly enforced on API endpoints
  • Look for sequential IDs that can be enumerated (orders, users, carts)
